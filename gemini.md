# Briefly AI Context & Development Guide

본 문서는 AI 코딩 어시스턴트(Gemini 등)가 이 저장소를 분석하고 추가 개발을 진행할 때, 프로젝트의 맥락을 정확히 이해하고 오류 없는 코드를 작성할 수 있도록 돕는 시스템 가이드라인입니다.

---

## 1. 프로젝트 목적 및 핵심 파이프라인
Briefly는 RSS 기반 뉴스 수집 및 AI 요약 대시보드 자동 배포 파이프라인입니다.
1. **수집 (`crawler.py`)**: 카테고리별(Tech, Politics, International) RSS 피드에서 최신 뉴스를 수집합니다.
2. **요약 (`summarizer.py`)**: Groq LLM API를 사용하여 기사 내용을 엄격한 규칙 하에 한글 3줄로 요약합니다.
3. **빌드 & 백업 (`app.py`)**: 기존 `index.html`을 `history/` 폴더에 타임스탬프 이름으로 복사하고, 새로 렌더링된 다크 테마 HTML로 덮어씁니다.
4. **배포 (`.github/workflows/update_news.yml`)**: 1시간 주기 Actions 실행 시 웹 페이지에 직접 호스팅 정적 리소스를 배포합니다.

---

## 2. AI 어시스턴트가 반드시 준수해야 할 코딩 가이드라인

### A. 엄격한 XSS 방어 및 보안 가이드
신뢰할 수 없는 외부 RSS 피드 입력을 그대로 웹 브라우저에 서빙하기 때문에, XSS 위협 차단이 최우선 사안입니다.
* **Jinja2 자동 이스케이프 유지 (`app.py`)**:
  Jinja2 `Environment` 생성 시 `select_autoescape(['html', 'xml'])`를 필수 적용해야 하며, 템플릿 변수에 임의로 `| safe` 필터를 사용하지 마십시오.
* **URL 프로토콜 스키마 검증 (`crawler.py`)**:
  수집한 뉴스 원문 주소(`article['link']`)는 반드시 `http://` 또는 `https://`로 시작하는지 검증하는 구문을 거쳐야 합니다. `javascript:` 스키마를 활용한 XSS 우회를 사전에 차단합니다.

### B. AI 요약 포맷 가드레일 (`summarizer.py`)
현재 요약 엔진은 Groq Cloud API의 `llama-3.3-70b-versatile` 모델을 활용하며 `temperature`는 `0.1`로 제어합니다.
* **시스템 프롬프트 제약**: 요약은 반드시 정확히 3개 이내의 불렛 포인트(•)여야 하며, 종결 어미는 '~함', '~임' 등의 명사형이어야 합니다. 사족(설명, 안내 텍스트)은 일절 금지됩니다.
* **파이썬 후처리 필터**: LLM의 응답 이탈을 보완하기 위해, 줄바꿈 단위로 응답을 파싱한 뒤 오직 `•` 문자로 시작하는 행만 필터링하여 합쳐 반환하는 가드레일 코드를 유지하십시오.

### C. 데이터 백업 로직의 원칙 (`app.py`)
`index.html` 생성 작업 이전에 기존 파일의 마지막 수정 시간(mtime)을 파일명으로 활용하여 `history/index_YYYYMMDD_HHMMSS.html` 형태로 반드시 파일 복사 백업 작업을 선행해야 합니다.

## 3. Git 워크플로우 및 자동화 배치
* **이중 브랜치 전략**: 
  * `main`: 라이브 사이트 배포 전용 브랜치 (직접 배포 동작)
  * `develop`: 모든 개발 사항이 통합되는 중심 개발 브랜치
  * `feature/...`: 기능 단위 독립 격리 개발용 브랜치
* **동적 푸시 스크립트 (`push.bat`)**:
  현재 로컬의 체크아웃된 브랜치명을 런타임에 동적으로 조회(`git branch --show-current`)하여 해당 브랜치로만 안전하게 pull rebase 및 push를 통합 처리하는 쉘 구조를 취하고 있습니다. 개별 수동 푸시 대신 배치 파일 활용을 권장합니다.
* **커밋 메시지 작성법**:
  커밋 메시지는 (버전) - (변경사항) 형식으로 씁니다.
  예 : Alpha 0.0.1 - change
